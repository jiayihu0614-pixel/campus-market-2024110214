4. 遇到的问题与解决方法
代码完成后，我第一次逐个访问路由时，页面显示无法连接。开始时我怀疑是路由路径或组件导入错误，但 TypeScript 检查和生产构建都已经通过。继续检查后发现，真正原因是原来的 Vite 开发服务器已经停止，并不是路由代码出错。重新启动开发服务器后，我依次访问 `/`、`/trade`、`/lost-found`、`/group-buy`、`/errand`、`/publish`、`/message` 和 `/user`，8 个路径都能够正常返回和显示。

另外，种子首页原来存在中文乱码。搭建新页面时，我重新填写了正常的中文标题和说明，并检查了文件编码。最后运行 `vue-tsc`、ESLint、Oxlint 和 Vite 构建，确认代码没有明显的类型错误和编译错误。

5. 今日反思
通过今天的任务，我理解了页面骨架、路由导航和公共布局之间的关系。页面骨架先确定每个业务页面负责什么内容，路由导航建立页面之间的访问关系，公共布局则保证所有页面共享统一的顶部区域、导航和内容容器。先把这些基础结构搭清楚，后续增加商品列表、发布表单、聊天消息和个人数据时，就可以在对应页面中继续开发，而不需要反复调整整个项目。
