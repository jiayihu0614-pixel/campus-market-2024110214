# Day3 Evidence：Mock 数据建模与列表渲染

## 1. 今日完成内容

今天我在 Day2 已完成页面骨架、路由导航和公共布局的基础上，继续完成了校园轻集市 Day3 的数据驱动改造。我安装了 `axios` 和 `json-server`，在项目根目录创建 `db.json`，并增加 `pnpm mock` 启动命令。随后按业务拆分请求模块，新增通用信息卡片和空状态组件，最后将二手交易、失物招领、拼单搭子、跑腿委托四个页面从静态说明改成了接口数据的列表渲染。

## 2. Mock 数据设计说明

我把校园轻集市的核心信息理解为四类：二手交易关注商品标题、价格、分类、成色和交易地点；失物招领关注寻物或招领类型、物品、发生时间、地点和联系方式；拼单搭子关注活动类型、目标人数、当前人数和截止时间；跑腿委托关注任务类型、取送地点、酬劳和完成时间。每类 Mock 数据各准备了 5 条，内容使用教材、耳机、校园卡、奶茶拼单、取快递等真实校园场景，没有使用“test”或“商品1”之类的无意义内容。

## 3. 我的设计

我认为 Day3 的数据模型首先要服务列表页面，而不是一次性模拟完整数据库。因此我保留标题、描述、地点、时间、发布人和状态等共同展示字段，再为不同业务保留少量专属字段。例如二手商品保留 `price` 和 `condition`，拼单保留 `currentCount` 与 `targetCount`，跑腿保留 `pickupLocation`、`deliveryLocation` 和 `reward`。这样既能体现业务区别，又不会让结构过度复杂。

## 4. AI 设计

本次使用 Codex 辅助检查项目结构、安装依赖、生成 `db.json` 初稿、Axios API 模块和 Vue 列表渲染代码。AI 设计的优点是能快速统一字段命名，并覆盖四类业务需要的主要展示信息。不过 AI 也可能把数据结构扩展成用户表、订单表、评论表、收藏表或权限关系，这些内容会增加理解和调试成本，也超出了 Day3 范围。

## 5. 人工审查与调整

我对 AI 生成内容进行了人工审查。最终保留了实验手册要求的四个数据集合和核心字段，删除或没有采用登录 Token、订单编号、评论数量、收藏关系、审核权限等复杂设计。我还逐项核对了页面字段映射：跑腿的 `reward` 映射为卡片价格区域，取件和送达地点合并显示；拼单人数放入描述；失物招领根据 `lost` 和 `found` 显示“寻物”或“招领”标签。人工检查还包括 JSON 格式、文件路径、路由是否保持不变，以及 Day1、Day2 文件是否被误删。

## 6. JSON Server 接口模拟

项目通过根目录的 `db.json` 和 JSON Server 模拟本地接口，端口为 `3001`。启动后可以访问 `/trades`、`/lostFounds`、`/groupBuys` 和 `/errands`。前端服务与 Mock 服务需要分别启动，如果只启动 Vite，页面会提示确认 JSON Server 是否已经运行。Day3 使用本地接口的目的，是练习前后端数据流，不是真实后端开发。

## 7. Axios 请求封装

我在 `src/api/http.ts` 中创建了基础 Axios 实例，统一设置 `baseURL` 和超时时间；然后按业务拆分 `trade.ts`、`lostFound.ts`、`groupBuy.ts`、`errand.ts`。这样页面只调用对应的查询函数，不需要在每个页面重复写接口地址，也没有加入 JWT、复杂拦截器或登录跳转。

## 8. 页面列表渲染

四个核心页面都使用 `ref` 保存数组，在 `onMounted` 中请求接口，并使用 `v-for` 渲染 `ItemCard.vue`。当数组为空或接口无法访问时，使用 `EmptyState.vue` 显示提示。页面内容因此不再写死在 Vue 模板中，而是由接口返回数据驱动。这样以后修改 `db.json` 或替换真实接口时，页面结构不需要重新编写。

## 9. 遇到的问题

安装环境时发现新版测试版 `json-server` 已取消实验手册中的 `--watch` 参数。为了保持课堂命令一致，我改用支持该参数的稳定版本。另一个需要注意的问题是 `db.json` 必须是严格 JSON，不能写注释，也不能在最后一个字段后保留多余逗号。前端和 Mock API 还必须同时运行，否则 Axios 无法取得数据。

## 10. 后续计划

后续可以在下一阶段增加必要的加载状态、简单筛选和发布表单，并继续复用目前的 API 分层和通用卡片。Day3 暂时不实现复杂登录、订单、评论、收藏和消息系统，因为本阶段的核心目标是理解 Mock 数据、JSON Server、Axios 请求与页面列表渲染之间的关系，先把基础数据流跑通。
